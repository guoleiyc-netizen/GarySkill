## 🎯 v3.3 核心更新

### 1. 案主隐私保护机制（⭐⭐⭐ 核心更新）

#### 1.1 强制脱敏处理

**更新目的**：
保护对话参与者的隐私，避免可识别信息泄露。

**脱敏处理清单**：

| 原文类型 | 脱敏处理 | 示例 |
|---------|---------|------|
| 人名 | 泛化处理或代称 | "张老师" → "一位英语老师" |
| 公司名称 | 去除或泛化 | "某科技公司" → "某公司" |
| 学校名称 | 去除或泛化 | "射阳中学" → "某中学" |
| 具体项目 | 去除或泛化 | "数字化转型项目" → "某项目" |
| 职位称谓 | 泛化处理 | "COO/CEO" → "管理岗位" |
| 行业特征 | 去除强特征词汇 | "拟IPO" → "正在快速发展" |
| 具体时间 | 模糊化 | "2026年1月23日" → "近期" |
| 具体数字 | 模糊化 | "两个月交期" → "非常紧张的交期" |

#### 1.2 脱敏检查清单

**发布前必检项目**：

- [ ] 是否有公司名称、品牌名
- [ ] 是否有具体项目名称
- [ ] 是否有具体人名、职务名
- [ ] 是否有强行业特征词汇
- [ ] 是否有可识别的时间地点信息
- [ ] 确保案主完全无法被识别

#### 1.3 双版本管理

**版本类型**：

1. **原版**：
   - 保留完整细节
   - 适合私密分享
   - 不公开发布

2. **脱敏版**：
   - 去除所有可识别信息
   - 适合公开发布
   - ⭐ 推荐使用

**文件命名规范**：

```
文章标题-原版.md
文章标题-脱敏版.md
```

   - 配色方案

3. **关键词/标签**（5-8个）

4. **发布时间建议**
   - 最佳时间

   - 避开时间

5. **朋友圈推广话术**（2-3个）

## 🔄 工作流程更新

### 阶段7后新增：阶段8 - 隐私保护与版本管理

#### 8.1 脱敏处理

**强制执行**：

1. 识别所有可识别信息
2. 应用脱敏处理清单
3. 生成脱敏版文章
4. 保留原版作为记录

#### 8.2 版本管理

**文件组织**：

```
项目文件夹/
├── 原版/
│   ├── 第1篇-标题-原版.md
│   └── 第2篇-标题-原版.md
├── 脱敏版/
│   ├── 第1篇-标题-脱敏版.md
│   └── 第2篇-标题-脱敏版.md
└── 发布准备包.md
```

#### 8.3 社会责任感检查

**必检项目**：

- [ ] 是否涉及心理危机？
  - 是 → 添加心理援助信息
  - 否 → 跳过

- [ ] 是否可能引发负面情绪？
  - 是 → 引导积极讨论
  - 否 → 正常发布
